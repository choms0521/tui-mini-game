# Tron (Light Cycles) 개발 계획서

작성일: 2026-06-23
대상 저장소: `tui-mini-game`
근거 문서: [`docs/game-candidates.md`](../../game-candidates.md) · [`docs/plans/tier1-games-plan.md`](../tier1-games-plan.md)

폴더: `tron/` · 브랜치: `feat/game-tron`

> 공통 구현 규약(불변 상태, 주입 RNG, 렌더 분리, 영어 식별자, 파일 경계)은
> [`tier1-games-plan.md` §2](../tier1-games-plan.md)를 따른다. 본 문서는 게임 고유 스펙만 다룬다.

## 1. 개요

- **장르**: 실시간 액션
- **재사용 진법**: Snake 변형(이동 + 자취) + 적 AI
- **한 줄 소개**: 두 라이트사이클이 자취(벽)를 남기며 달리고, 먼저 벽·자취·경계에 충돌하는 쪽이 패배.
- **재미 포인트**: 상대를 좁은 공간으로 몰아넣는 순간 판단. 짧고 강렬한 대결.

## 2. 규칙

- 격자 위에서 사람·AI 사이클이 **매 틱 자동 전진**하며 지나온 칸을 벽으로 남긴다.
- 벽(자기/상대 자취), 격자 경계, 상대와의 동시 진입 칸에 부딪히면 사망.
- 둘이 같은 칸에 동시에 진입하면 양측 사망 → 무승부. 한쪽만 살아남으면 그쪽 승리.
- 진행 방향은 바꿀 수 있으나 180° 역주행은 불가.

## 3. 재사용 자산

- 이동/자취/충돌 골격: Snake `game.py`의 본체 전진·자기 충돌 로직을 "벽 집합 + 두 주체"로 확장.
- 실시간 입력 루프: Pong/Snake `main.py`의 틱 기반 비차단 입력 패턴.

## 4. 상태 모델 (`@dataclass(frozen=True)`)

```
GameState:
  walls:        frozenset[Pos]     # 양측 자취 전체
  player_pos:   Pos
  player_dir:   Pos                # (드로우, 디칼럼) 단위 벡터
  ai_pos:       Pos
  ai_dir:       Pos
  player_alive: bool
  ai_alive:     bool
  game_over:    bool
  winner:       int | None         # 0=무승부
  tick:         int
```

- 게임 로직(`tick(state)`)은 blessed에 의존하지 않는 순수 함수로 두어 헤드리스 테스트가 가능하게 한다.

## 5. AI 전략

- **회피 우선**: 다음 후보 방향 중 즉사하지 않는 방향만 추리고, 그중 **도달 가능한 빈 공간(flood-fill 면적)이 가장 큰** 방향을 선택.
- 동률 면적은 주입 RNG로 tie-break. 모든 방향이 치명적이면 임의 합법 방향(사망 불가피).

## 6. 입력

- `화살표`: 사람 사이클 방향 전환(180° 역주행 무시)
- `p`: 일시정지 · `r`: 재시작 · `q`: 종료
- 입력과 무관하게 틱마다 양측 전진(실시간).

## 7. 렌더

- 격자 + 경계, 사람 자취/머리(한 색), AI 자취/머리(다른 색).
- 패널: 조작법, 틱/생존 상태, 승/무 배너.
- Snake/Pong과 동일한 깜빡임 없는 전체 프레임 1회 출력.

## 8. 셀프테스트 필수 항목

- 틱 전진: 머리가 방향대로 이동하고 직전 칸이 벽으로 남음.
- 충돌 사망: 벽/자취/경계 진입 시 해당 사이클 사망.
- 정면 충돌: 같은 칸 동시 진입 → 양측 사망 → 무승부.
- 180° 역주행 입력 무시.
- AI는 즉사하지 않는 방향이 있으면 그 방향을 반환(회피 동작).
- 한쪽 생존 시 승자 판정.
- 불변성, 주입 RNG로 AI 결정적, `tick()` 순수성(루프와 분리).
