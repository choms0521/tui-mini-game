## 1. 개요

- **장르**: 보드 + AI
- **재사용 진법**: 그리드 렌더 + 미니맥스(Connect Four에서 도입한 알파-베타 사고)
- **한 줄 소개**: 8x8 보드에서 상대 돌을 양끝으로 둘러싸 뒤집어, 마지막에 돌이 많은 쪽이 승리.
- **재미 포인트**: 한 수로 판이 뒤집히는 반전. 모서리 선점 등 위치 전략의 깊이.

## 2. 규칙

- 8x8. 중앙 4칸을 흑/백 2개씩 교차 배치한 상태로 시작.
- 합법 수: 어느 한 방향(8방향) 이상에서 상대 돌을 1개 이상 연속으로 끼워 자기 돌로 끝나는 자리. 둘 수 있는 모든 방향의 끼인 돌을 뒤집는다.
- 합법 수가 없으면 **패스**. 양쪽 모두 둘 수 없으면(또는 보드가 가득) 게임 종료.
- 종료 시 돌 수가 많은 쪽이 승리(같으면 무승부).

## 3. 재사용 자산

- 격자 렌더/커서 강조: Minesweeper/Connect Four `render.py` 패턴.
- 미니맥스 탐색·평가 골격: Connect Four의 알파-베타 구조를 8x8/뒤집기 평가로 일반화. (보드+AI 공통 미니맥스 유틸 추출 후보 — `game-candidates.md` §7)

## 4. 상태 모델 (`@dataclass(frozen=True)`)

```
GameState:
  board:          tuple[tuple[int, ...], ...]   # 0=빈칸, 1=흑, 2=백
  current_player: int                            # BLACK / WHITE
  game_over:      bool
  winner:         int | None                     # 0=무승부
  last_move:      Pos | None
  cursor:         Pos
```

- 합법 수 목록과 뒤집기 결과는 순수 함수로 유도(상태에 저장하지 않음).

## 5. AI 전략

- 알파-베타 미니맥스, 깊이 제한(예: 4~5).
- 평가 함수: `돌 수 차이` + `위치 가중치`(모서리 +높음, X/C 자리 −) + `합법 수(mobility) 차이`.
- 종반에는 평가를 단순 돌 수 차이로 전환(가능 시 완전 탐색). 동률 수는 주입 RNG로 tie-break.

## 6. 입력

- `화살표`: 커서 이동
- `Enter`/`Space`: 커서 자리에 착수(합법 수만 허용)
- `r`: 재시작 · `q`: 종료
- 사람=흑 선공, AI=백.

## 7. 렌더

- 8x8 초록 보드, 흑/백 트루컬러 돌.
- 사람 차례에는 합법 수 위치를 점 마커로 표시, 커서 강조.
- 패널: 양측 돌 수, 현재 차례, 패스 알림, 승/무 배너.

## 8. 셀프테스트 필수 항목

- 초기 4돌 배치 정확.
- `legal_moves`: 끼우기 성립 자리만 반환, 비성립 자리는 거부.
- 착수 시 관련 8방향의 끼인 돌이 모두 뒤집힘(다방향 동시 뒤집기 케이스 포함).
- 합법 수 없을 때 패스, 양측 패스/보드 가득 시 종료.
- 돌 수로 승자 판정(무승부 포함).
- AI는 합법 수 반환, 모서리가 비면 모서리를 선택(위치 가중치 동작).
- 불변성, 주입 RNG로 AI 결정적.
