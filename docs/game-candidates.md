# 터미널 게임 후보 발굴 (Game Candidates)

작성일: 2026-06-23
대상 저장소: `tui-mini-game`
문서 성격: 후보 큐레이션 (재미 · 아키텍처 적합도 · 구현 비용 기준 평가 및 우선순위)

## 1. 배경과 목적

현재 저장소에는 이미 10종의 터미널 게임이 동일한 진법으로 구현되어 있다.

> Tetris · Breakout · Snake · 2048 · Minesweeper · Space Invaders · Roguelike · Wordle · Sokoban · Pong

이 문서는 "다음에 만들면 재미있을 게임"을 4개 장르에 걸쳐 폭넓게 발굴하고, 이미 존재하는 게임과 중복되지 않으면서 우리 코드 구조에 잘 맞는 후보를 골라 우선순위를 매긴 결과를 정리한다. 목표는 "어떤 게임을 다음에 구현할지" 의사결정을 빠르게 내리는 것이다.

## 2. 아키텍처 제약과 재사용 자산

모든 기존 게임은 폴더 하나에 다음 구성을 따른다. 신규 게임도 이 패턴을 그대로 따르는 것을 전제로 적합도를 평가했다.

| 파일 | 역할 |
| --- | --- |
| `game.py` / `board.py` | 불변(immutable) 게임 상태와 규칙 로직 |
| `render.py` | blessed 기반 깜빡임 없는 트루컬러 렌더링 |
| `main.py` | blessed 입력 루프(엔트리) |
| `selftest.py` | 헤드리스 자동 테스트 |
| `meta.json` | 런처 자동 등록용 메타데이터 |
| `levels.py` / `words.py` / `solver.py` (선택) | 데이터/에셋 또는 탐색 로직 |

핵심 제약: 단일 터미널 화면, 키보드 입력, 불변 상태 갱신, 헤드리스 테스트 가능. 이 제약에 잘 맞을수록 적합도 점수가 높다.

재사용 가능한 기존 진법(패턴):

- **그리드 렌더 + 커서 이동**: Minesweeper, 2048, Sokoban
- **추측-피드백 루프**: Wordle (검정/노랑 힌트 계산)
- **탐색/풀이 알고리즘**: Sokoban `solver.py` (백트래킹/탐색)
- **실시간 게임 루프 + 충돌**: Pong, Breakout, Space Invaders
- **엔티티/맵 + 적 AI**: Roguelike

## 3. 평가 기준 (Rubric)

- **재미 (Fun)**: 별 1~5개. 터미널 환경에서의 체감 재미와 반복 플레이성.
- **적합도 (Fit)**: 별 1~5개. 단일 화면 + 불변 상태 + 헤드리스 테스트 패턴과의 정합성, 기존 진법 재사용 가능성.
- **구현 비용 (Effort)**: 낮음 / 중간 / 높음. 규칙 복잡도, AI/생성기 필요 여부, 렌더 난이도 종합.

추천 우선순위는 기본적으로 **재미 + 적합도가 높고 구현 비용이 낮은** 순서를 따른다.

## 5. 티어별 상세

### 1순위 — 즉시 착수 권장

#### Mastermind (코드브레이커)
- **한 줄 소개**: 숨겨진 색/숫자 조합을 제한된 시도 안에 추론해 맞추는 게임.
- **재미 포인트**: Wordle를 좋아하면 곧장 빠져드는 추론 재미. 한 판이 짧고 반복성이 높다.
- **적합도**: Wordle의 추측-피드백 루프를 거의 그대로 재사용한다. 차이는 피드백 형태("위치까지 맞음"/"색만 맞음" 개수)뿐이다.
- **핵심 난관**: 피드백 계산 정확도. 같은 색 중복 처리에서 실수하기 쉬우나 헤드리스 테스트로 쉽게 검증된다.
- **데이터**: 없음(코드 무작위 생성).

#### Connect Four (사목)
- **한 줄 소개**: 7x6 보드에 번갈아 말을 떨어뜨려 가로/세로/대각선 4개를 먼저 잇는 쪽이 승리.
- **재미 포인트**: 규칙이 단순하면서 수읽기 긴장감이 크다. AI 대전과 2인 대전 모두 가능하다.
- **적합도**: 그리드 렌더는 Minesweeper/2048에서, 미니맥스(알파-베타 가지치기) 탐색은 Sokoban solver의 탐색 사고에서 빌려올 수 있다. 보드+AI 장르의 첫 도입으로 이후 Reversi/Gomoku 자산의 토대가 된다.
- **핵심 난관**: AI 난이도 밸런싱(탐색 깊이 제한)과 4방향 승리 판정.
- **데이터**: 없음.

#### Sudoku
- **한 줄 소개**: 9x9 격자에 1~9를 규칙에 맞게 채우는 대표적 두뇌 퍼즐.
- **재미 포인트**: 스테디셀러. 난이도 조절로 무한한 반복 플레이성을 제공한다.
- **적합도**: 9x9 그리드 + 커서 이동은 Minesweeper/Sokoban 방식 그대로다. 백트래킹 풀이/검증기는 Sokoban `solver.py` 패턴과 잘 맞는다.
- **핵심 난관**: 유일해를 보장하는 퍼즐 생성기(백트래킹 생성 후 난이도별 칸 제거). 가장 큰 비용 요소다. 정적 퍼즐 세트로 시작하면 비용을 낮출 수 있다.
- **데이터**: 생성기 또는 정적 퍼즐 세트.

### 2순위 — 충분히 가치 있는 후보

#### Battleship
- **소개/재미**: 가려진 적 함대를 사격으로 추적해 격침. 추론과 운의 긴장감.
- **적합도/난관**: 내 보드/적 보드 두 개의 그리드는 Minesweeper 렌더 재사용. AI를 확률 기반 hunt/target 전략으로 만들면 재미가 크게 올라간다(난관). 함선 배치 검증 필요.

#### Reversi (Othello)
- **소개/재미**: 8x8 보드에서 상대 돌을 양끝으로 둘러싸 뒤집기. 한 수로 판이 뒤집히는 반전 재미.
- **적합도/난관**: 그리드 + 미니맥스에 위치 가중치 평가 함수. 합법 수 계산과 8방향 뒤집기 판정이 핵심.

#### Gomoku (오목)
- **소개/재미**: 큰 격자에 번갈아 두어 5개를 먼저 잇기.
- **적합도/난관**: 5목 스캔은 쉽지만, 강한 AI는 위협 패턴 점수 휴리스틱이 필요(비용 상승 지점). 휴리스틱 수준을 타협하면 중간 비용에 수렴.

#### Tron (Light Cycles)
- **소개/재미**: Snake 변형. 지나간 자리에 벽이 남고, 먼저 충돌하는 쪽이 패배. 적 AI 또는 2인 대전.
- **적합도/난관**: Snake 코드를 직접 변형 가능. 적 AI의 회피 로직(빈 공간 우선 탐색)이 난관.

#### Frogger
- **소개/재미**: 차선과 강을 건너 반대편에 도달. 타이밍 액션.
- **적합도/난관**: 차선별 이동 장애물 + 실시간 루프는 Space Invaders/Pong에서 재사용. 충돌/탑승(통나무) 판정이 핵심.

#### Blackjack
- **소개/재미**: 딜러와의 21 맞추기. 베팅과 히트/스탠드 판단.
- **적합도/난관**: 딜러는 고정 전략(17 이상 스탠드)이라 AI 부담이 적다. 카드 ASCII 렌더와 에이스(1/11) 처리가 포인트.

### 빠른 승리 (Quick Wins) — 저비용으로 라인업 보강

- **15-Puzzle**: 4x4 타일 슬라이딩. 2048과 구조가 거의 같아 가장 빠르게 만들 수 있다. 가해(풀 수 있는) 상태 보장 셔플만 주의.
- **Lights Out**: 5x5 토글 그리드. 무작위 클릭으로 가해 보드를 생성하므로 매우 단순하다.
- **Flappy (one-button)**: 중력 + 스크롤 장애물. Breakout/Pong의 실시간 루프 재사용. 물리 파라미터 튜닝이 재미를 좌우한다.

### 탐색 / 도전 — 재미는 크나 비용이 큰 후보

- **Nonogram (Picross)**: 행/열 단서로 그림을 그려내는 퍼즐. 그림이 드러나는 쾌감이 크지만 퍼즐 생성 + 단서 계산 + 해 검증이 필요(중간~높음).
- **Pac-Man**: 미로 자체는 쉽지만 4마리 유령의 추격/산개 모드 AI와 파워펠릿 상태 전환이 복잡하다. Roguelike의 맵/엔티티 사고를 빌릴 수 있다.
- **Solitaire (Klondike)**: 테이블로/파운데이션/스톡의 상태가 크고 카드 이동 규칙이 많다. 카드 ASCII 렌더 부담도 있다.
- **Asteroids**: 회전/관성 물리를 문자 그리드에 벡터로 표현하는 것이 까다롭다. 실시간 루프는 재사용 가능.
- **Chess**: 캐슬링/앙파상/승진 등 규칙과 체크메이트 판정, 그리고 AI까지 모두 무거워 매우 높은 비용. 재미는 최상급.
